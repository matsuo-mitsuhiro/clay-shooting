## 2. 公開エリア詳細

### 2.1 トップページ（`/`）

```
┌─────────────────────────────────────────┐
│        クレー射撃大会 運営システム      │
├─────────────────────────────────────────┤
│  [ 大会一覧 ] [ 過去の大会 ]            │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ 大阪府民スポーツ大会            │    │
│  │ [トラップ]              [概要]  │    │
│  │ 📍 大阪総合射撃場              │    │
│  │ 📅 2026年5月6日                │    │
│  └─────────────────────────────────┘    │
│  ...                                    │
└─────────────────────────────────────────┘
```

**ボタン動的変更（v3.64〜）:**
- 点数なし・組未発表 → `[概要]` または `[概要・申込]`（申込期間中）
- 組発表済・点数なし → `[射順]` または `[射順・申込]`
- 点数1件以上あり → `[成績]`（赤色）

判定基準: `score_count`、`squad_published_at`、`apply_start_at/apply_end_at`

タブ:
- **大会一覧**: 最終日が今日以降の大会（昇順）
- **過去の大会**: 最終日が今日より前の大会（降順）

> v3.78 で左下の隠し管理者ボタンは削除済。管理者は `/admin/login` から入る。

### 2.2 閲覧者：大会選択（`/viewer`）
公開大会の一覧表示。任意の大会をクリックで `/viewer/[id]` へ。

### 2.3 閲覧者：成績画面（`/viewer/[id]`）

```
┌────────────────────────────────────────────┐
│ ヘッダー：[QR][フィルター][↺更新]         │ ← sticky
├────────────────────────────────────────────┤
│ 順位 | 氏名 | 所属 | クラス | R1〜R8 | 合計  │ ← sticky thead
│  1   | ... | ... | ...   | ...    | 95   │
│ ...                                        │
│ ── 賞典外 ──                              │
│  -   | 田中 | 大阪 | A    | ...    | 80   │
└────────────────────────────────────────────┘
```

- 認証不要（任意で会員番号入力 → 自分の行をハイライト・最上部スクロール）
- フィルター・集計はモーダル（v3.41〜）
- 表示順: 通常選手（合計DESC順 + カウントバック）→ `── 賞典外 ──` 区切り → 賞典外（total DESC）
- 失格・棄権は末尾固定、rank=NULL
- レイアウトは `height:100vh; flex column`、スクロールコンテキストは1箇所のみ

### 2.4 申込フロー（`/tournaments/[id]/apply`）

```
[Step 1: 認証]
  会員番号入力（半角数字、player_master検索）
  メールアドレス入力
       │
       ▼ 「申込コードを送信」
  → registration_tokens（purpose='apply'）に6桁コード保存
  → メールで6桁コードを送信
       │
       ▼ 6桁コード入力
       │
[Step 2: 申込フォーム]
  氏名・所属・クラス（player_master から自動補完）
  審判フラグ
  参加日（day1/day2/both — 2日制大会のみ both 選択可）
       │
       ▼ 「申込を送信」
       │
[完了]
  確認メール送信
  続けて他大会へ申込（連続申込モード）
```

**残数チェック**: `max_participants - active な registrations` を 1日目・2日目別カウント（`both` は両方に消費）

**重複防止**: 同一 `tournament_id × member_code × status='active'` は禁止 → `registration_logs.log_type='duplicate_error'`

**player_master との同期**: 申込時のクラス・所属が player_master と異なれば `player_master.change_history` に追記

### 2.5 キャンセルフロー（`/tournaments/[id]/cancel`）

```
メールアドレス入力
   │
   ▼ ワンタイムURL（1時間有効）をメール送信
   │
URLクリック → /tournaments/[id]/cancel/[token]
   │
   ▼ 「キャンセルを確定」
   │
完了 → registrations.status='cancelled', cancelled_by='user'
       registration_logs.log_type='cancel_by_user'
```

> 申込キャンセル期限（`tournaments.cancel_end_at`）を過ぎると操作不可

### 2.6 FAQ（`/faq`）と お問合せ（`/support`）
- FAQ: `faq` テーブルから取得して表示
- お問合せ: メールアドレス + 質問を `support_questions` に保存、管理者へ通知メール

## 3. 認証エリア詳細

### 3.1 ログイン（`/admin/login`）

```
[運営管理者]                [システム管理者]
会員番号                    Googleでログイン →
パスワード                  （Google OAuth, SYSTEM_ADMIN_EMAILS 許可制）
[ログイン]
[パスワードをお忘れの方はこちら]
```

成功すると `operation_logs` に `action='login'` 記録。

### 3.2 招待登録（`/admin/register?token=...`）

システム管理者が `admin_invitations` で発行したトークンを介してアクセス。
会員番号・氏名・メール・パスワードを入力 → `tournament_admins` に登録 → 招待トークンを `used_at` でマーク。

### 3.3 パスワードリセット
1. `/admin/forgot-password`: 会員番号またはメール入力 → ワンタイムリンク送信（`password_reset_tokens`）
2. `/admin/reset-password/[token]`: 新パスワード入力 → `tournament_admins.password_hash` を更新

### 3.4 管理者：大会一覧（`/admin`）
```
┌─────────────────────────────────────┐
│ ＋新規大会作成 ／ 過去の大会をコピー  │
├─────────────────────────────────────┤
│ 大会カード一覧（運営管理者は自所属のみ）│
│  ・名前・種目・日付・申込状況       │
│  ・[管理する] ボタン                │
└─────────────────────────────────────┘
```

### 3.5 大会管理（`/admin/[id]`）— 8タブ構成

タブ切替は URL クエリ `?tab=...` で同期（共有可能）。
モバイル（768px未満）はハンバーガーメニュー＋ドロワー。

#### Tab 1: 大会設定
- 大会情報（名称・会場・日付・種目・セット番号）
- QRコード（管理者用・閲覧者用・運営管理者招待用）
- **危険ゾーン**: リセット（選手・点数削除）／削除（CASCADE）
- 大会ID表示（編集不可、v3.69〜）

#### Tab 2: 申込設定
- 募集人数・申込開始/終了日時・キャンセル期限
- 競技開始/開門/受付/練習クレー時刻
- 中止お知らせ方法・注意書き
- 組発表（squad_published_at, squad_comment）

#### Tab 3: 申込管理
- 申込一覧（インライン編集モード、v3.70〜）
- 編集可能項目: 氏名・所属・クラス・審判・参加日・**会員番号（v3.73〜）**
- ステータス: active / cancelled
- 操作: 参加キャンセル / 削除（手動のみ）/ 復元 / 移行（→members）
- 手動追加（source='manual'）

#### Tab 4: 選手管理
- 6行固定の組単位入力テーブル（最大6名/組）
- 1日目・2日目切替
- インライン編集（**所属と賞典外のみ編集可、会員番号/クラス/審判は表示のみ** v3.73〜）
- 「選手組替」ボタン: DnDで組内・組間移動
- 「ランダム組替」ボタン: 主催協会の審判を各組に配置 + 両日参加者を前方優先
- 「前日の成績順組替」ボタン（2日目のみ、v3.63〜）
- 「1日目からコピー」ボタン

#### Tab 5: 点数登録
- 1日目（R1-R4）／2日目（R5-R8）切替（1日制は非表示）
- 全組／組別フィルター
- セルクリック → テンキーポップアップ
  - 0〜22: 青色、23〜25: 赤色
  - 「決定」で次の選手へ縦移動（縦入力UX）
- CB入力（1〜6専用テンキー）／FR入力（1〜99）
- 成績ステータス: 有効／失格／棄権 ドロップダウン
- 点数保存時に operation_logs に記録（action='score_save'）

#### Tab 6: 成績確認
- v_results からの順位表
- クラス別フィルター（参加クラスのみドロップダウン表示）
- 通常選手 → 賞典外 → 失格・棄権 の順
- 手動順位設定モーダル（DnD並び替え、v3.43〜、賞典外を除外）
- CB/FR列は条件表示

#### Tab 7: 記録審査
- ルール・環境（天候・気温・風速）・担当者（審査委員長等）入力
- 「大会記録審査表Excel出力」ボタン
  → `templates/inspection-report-template.xlsx` に値を埋め込み（JSZip+XML方式、v3.35〜）

#### Tab 8: 大会報告
- ペア大会自動検出（同日・同協会のトラップ＋スキート）
- 参加人数: クラス別×種目別（registrations.status='active'から集計、v3.74〜）
- 奨励金獲得者: 50/75/100ストレート自動検出＋手動補正
- 本部納入金: 公認料・広告代・奨励金納付・参加料納付の自動計算
- 「大会報告書Excel出力」ボタン → `templates/report-template.xlsx`

### 3.6 マスター画面群

| URL | 内容 |
|-----|------|
| `/admin/admins` | 運営管理者の作成・編集・削除（システム管理者のみ）。招待QRコード生成も可能 |
| `/admin/players` | 選手マスター 1370名。検索・編集・追加・削除。trap_class / skeet_class 別管理 |
| `/admin/associations` | 47都道府県＋芸文の協会マスター。会長名・中止連絡方法・注意書き・関連射撃場 |
| `/admin/shooting-ranges` | 射撃場マスター（都道府県グループ表示） |
| `/admin/logs` | operation_logs を表示。所属・大会・アクションでフィルター |
| `/admin/support` | お問合せ管理（質問一覧・回答・FAQ・トークン発行） |

> 運営管理者は自所属の大会・選手のみアクセス可。システム管理者は全件アクセス可。

## 4. QRコードのリンク先

| QR種別 | リンク先 | 配置場所 |
|--------|---------|---------|
| 管理者用QR | `/admin/[id]` | 大会設定タブ（運営者間共有用） |
| 閲覧者用QR | `/viewer` | 大会会場掲示 |
| 申込用QR | `/tournaments/[id]/apply` | 申込告知（メール・SNS） |
| 運営管理者招待QR | `/admin/register?token=...` | システム管理者→運営管理者 |

## 5. レスポンシブ対応

- ブレークポイント: **768px**
- 768px未満（スマホ）:
  - AdminDetail: ハンバーガーメニュー + ドロワー

  - タブバー横並び
  - テーブル横スクロール
