## 環境変数（Vercel に設定済み）

### 必須環境変数（全8個）

| 変数名 | 値の例 | 用途 |
|--------|--------|------|
| `DATABASE_URL` | `postgresql://...neon.tech/...` | Neon PostgreSQL 接続文字列 |
| `NEXTAUTH_SECRET` | 32文字以上のランダム文字列 | JWTの署名鍵（`openssl rand -base64 32`） |
| `NEXTAUTH_URL` | `https://clay-shooting.vercel.app` | 本番URL（OAuth リダイレクト解決用） |
| `GOOGLE_CLIENT_ID` | `xxxxx.apps.googleusercontent.com` | Google OAuth クライアントID |
| `GOOGLE_CLIENT_SECRET` | `GOCSPX-...` | Google OAuth クライアントシークレット |
| `SYSTEM_ADMIN_EMAILS` | `matsuo@repros.co.jp` | システム管理者メール（カンマ区切りで複数可） |
| `EMAIL_USER` | `noreply@example.com` | 送信用Gmailアドレス |
| `EMAIL_PASS` | Gmailアプリパスワード | Gmailアプリパスワード（2段階認証必須） |

> ⚠️ 実値は `C:/projects/clay-shooting/開発メモ.txt` を参照

### ローカル開発用（`.env.local`）

プロジェクトルートに `.env.local` を作成:

```
DATABASE_URL=（開発メモ.txtの値）
NEXTAUTH_SECRET=（開発用に同じ値でも可）
NEXTAUTH_URL=http://localhost:3000
GOOGLE_CLIENT_ID=（同上）
GOOGLE_CLIENT_SECRET=（同上）
SYSTEM_ADMIN_EMAILS=matsuo@repros.co.jp
EMAIL_USER=（同上）
EMAIL_PASS=（同上）
```

## Google OAuth セットアップ

### Google Cloud Console 設定手順
1. https://console.cloud.google.com/ にアクセス
2. プロジェクト選択 → 「APIとサービス」→「認証情報」
3. 「OAuthクライアントIDを作成」→ アプリケーションの種類: **ウェブアプリケーション**
4. 承認済みのリダイレクトURI に追加:
   - **本番**: `https://clay-shooting.vercel.app/api/auth/callback/google`
   - **ローカル**: `http://localhost:3000/api/auth/callback/google`
5. クライアントID・クライアントシークレットを取得 → Vercel と `.env.local` に設定

### 許可メールの管理
- `SYSTEM_ADMIN_EMAILS` にカンマ区切りで設定（例: `a@x.com,b@y.com`）
- ここに無いメールでGoogleログインしても認証拒否

## Gmail SMTP セットアップ（メール送信）

### Gmail アプリパスワード取得手順
1. Gmailアカウントで2段階認証を有効化
2. https://myaccount.google.com/apppasswords にアクセス
3. アプリ名を「Clay-shooting」など任意で作成
4. 16文字のアプリパスワードを取得 → `EMAIL_PASS` に設定

### 用途
- 申込時のワンタイムコード送信（6桁）
- 申込完了確認メール
- キャンセル用ワンタイムURL送信
- パスワードリセット用URL送信
- お問合せ受信時の管理者通知

## デプロイ手順

### 通常のデプロイ
```bash
# 1. TypeScriptチェック（必須）
npx tsc --noEmit --incremental false

# 2. lib/version.ts のバージョン更新
# - 新機能・仕様変更: 3.x → 3.(x+1)
# - バグ修正: 3.x → 3.x.1, 3.x.1 → 3.x.2

# 3. docs 更新（CLAUDE.md 必須項目）
# - docs/05_開発履歴.md（新バージョン記録）
# - docs/07_AI引き継ぎメモ.md（主要機能一覧の更新）

# 4. コミット＆プッシュ
git add <変更ファイル>
git commit -m "変更内容の説明"
git push origin main
# → Vercelが自動でビルド・デプロイ（約2〜3分）
```

### デプロイ確認
- https://vercel.com のダッシュボードで進行状況確認
- 本番URL（https://clay-shooting.vercel.app/）のフッターでバージョン番号確認
- 反映タイムラグは通常 2〜3 分

## DBセットアップ

### 初回セットアップ
1. Neonダッシュボード → プロジェクト「Clay-shooting」を選択
2. 「SQL Editor」を開く
3. `db/schema.sql` を実行（初期テーブル作成）
4. `db/migrations/001〜023.sql` を順番に実行

### 新規 migration 追加時
1. `db/migrations/NNN_xxx.sql` を作成（連番、ゼロパディング）
2. **Node.js + @neondatabase/serverless で自動実行**
   ```bash
   # 例: scripts/run-migration.js を作成して
   node scripts/run-migration.js db/migrations/024_xxx.sql
   ```
3. ⚠️ **memory feedback (Neon migration)**: ユーザーに依頼せず Claude が自分で実行する
4. 実行後、`docs/05_開発履歴.md` に migration 番号と実行済の旨を明記

### Migration 実行履歴（023まで全実行済み）

| No. | 概要 | 状態 |
|-----|------|------|
| 001〜023 | 詳細は `docs/03_DBスキーマ.md` の一覧参照 | ✅ Neon実行済 |

## Windowsローカル固有の問題

### EPERM エラー（`next build` 失敗時）
```bash
node windows-eperm-fix.js
```
> ただし根本解決にはならない場合あり。Vercelでのビルドは正常動作。

### TypeScript チェック単独実行
```bash
npx tsc --noEmit --incremental false
```
- 出力なし = エラーなし（コミット可）
- エラー1件でもあれば修正必須

## トラブルシューティング

### NextAuth でログインできない
- `NEXTAUTH_SECRET` 設定漏れ → 32文字以上のランダム文字列を Vercel に設定
- `NEXTAUTH_URL` が本番URLと不一致 → 一致させる
- Google OAuth リダイレクトURI が設定されていない → Google Cloud Console で追加

### メール送信失敗
- `EMAIL_PASS` がアカウントパスワードになっている → アプリパスワードに変更
- Gmail送信制限（1日500通）に到達 → 監視

### `/admin/*` にアクセスして即ログイン画面に飛ぶ
- セッション切れ（JWT有効期限） → 正常動作
- middleware.ts の matcher が `/admin/:path*` を含むことを確認
