## 重要な設計上の注意点

### 1. PostgreSQL数値型→文字列問題
```typescript
// ❌ ダメ（文字列にtoFixedはない）
r.average?.toFixed(2)

// ✅ 正しい
Number(r.average).toFixed(2)
```
`v_results` ビューのNUMERIC集計値はJavaScriptで文字列として届く。

### 2. member_code が紐付けキー
- `members` テーブルと `scores` テーブルは `member_code` で紐付け（外部キーなし）
- `member_code` が NULL の選手は点数登録不可（UIで警告あり）

### 3. Next.js App Router の params
```typescript
// params は Promise<> でラップされている
const { id } = await params;  // ← await 必須
```

### 4. Windowsローカルビルドについて
- `next build` はEPERMエラーで失敗する場合がある（Windows固有）
- TypeScriptチェック: `npx tsc --noEmit --incremental false`（エラーなしなら問題なし）
- 実際のビルドはVercel（Linux）が行うため、pushすれば自動で確認できる

## Vercel環境変数（認証システム用）

管理者認証を動作させるには、Vercel ダッシュボードで以下の環境変数を設定する必要があります。

| 変数名 | 値 | 説明 |
|--------|-----|------|
| `NEXTAUTH_SECRET` | ランダム文字列32文字以上 | JWTの署名鍵。`openssl rand -base64 32` で生成 |
| `NEXTAUTH_URL` | `https://clay-shooting.vercel.app` | 本番URL |
| `GOOGLE_CLIENT_ID` | Google Cloud Consoleで取得 | Google OAuth クライアントID |
| `GOOGLE_CLIENT_SECRET` | Google Cloud Consoleで取得 | Google OAuth クライアントシークレット |
| `SYSTEM_ADMIN_EMAILS` | `matsuo@repros.co.jp` | Googleログイン許可メール（カンマ区切りで複数可） |

### Google Cloud Console の OAuth設定手順
1. https://console.cloud.google.com/ にアクセス
2. プロジェクト選択 → 「APIとサービス」→「認証情報」
3. 「OAuthクライアントIDを作成」→ アプリケーションの種類：**ウェブアプリケーション**
4. 承認済みのリダイレクトURI に追加：
   - `https://clay-shooting.vercel.app/api/auth/callback/google`
5. クライアントID・クライアントシークレットをコピーしてVercelに設定
