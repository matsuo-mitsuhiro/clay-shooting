# クレー射撃大会 成績管理システム — システム概要（v3.78 時点）

## 目的
クレー射撃大会（トラップ・スキート）の選手管理・点数入力・成績閲覧・申込受付・大会報告までをワンストップで提供する Web アプリケーション。
管理者がタブレット等で点数を入力し、選手・観客が QR コードで成績をリアルタイム閲覧できる。

### `/admin/[id]` の8タブ構成（[components/admin/AdminDetail.tsx](../components/admin/AdminDetail.tsx) で定義）
1. **大会設定** — 大会情報・QRコード・危険ゾーン（リセット・削除）
2. **申込設定** — 募集人数・申込日時・スケジュール・中止連絡方法・注意書き
3. **申込管理** — 申込一覧・編集・キャンセル・移行・手動追加
4. **選手管理** — 組・番号配置（インライン編集・DnD組替・ランダム組替）
5. **点数登録** — テンキーポップアップ・賞典外・失格/棄権・CB/FR
6. **成績確認** — 順位・合計・平均・クラス別フィルター・手動順位
7. **記録審査** — ルール・環境情報・担当者・記録審査表Excel出力
8. **大会報告** — ペア大会対応・参加人数・奨励金・本部納入金・大会報告書Excel出力

## 役者と権限

| ロール | 認証方式 | 権限範囲 |
|-------|---------|---------|
| **システム管理者** | Google OAuth（`SYSTEM_ADMIN_EMAILS` で許可制） | 全大会・全機能。運営管理者の作成・削除も可 |
| **運営管理者** | 会員番号 + bcryptパスワード | 自所属協会の大会のみ。点数登録済の大会は削除不可 |
| **選手・申込者** | 会員番号 + メール（ワンタイムコード） | 自分の申込のみ |
| **閲覧者** | 認証不要（任意で会員番号入力 → 自分の行ハイライト） | 公開大会の成績閲覧 |

- TypeScriptチェックは `npx tsc --noEmit --incremental false` のみで可

## 既知の重要な設計上の注意点

### 1. PostgreSQL NUMERIC → JS 文字列問題
`v_results.average` などの NUMERIC は文字列で返る。
```typescript
// ❌ ダメ
r.average?.toFixed(2)
// ✅ 正しい
Number(r.average).toFixed(2)
```

### 2. member_code が選手↔点数の紐付けキー
- 外部キー制約なし
- `member_code` が NULL の選手は点数登録不可（UIで警告）

### 3. Next.js App Router の params は Promise
```typescript
const { id } = await params;  // ← await 必須
```

### 4. 所属名は短縮形で保存・正式名で表示
- DB保存: `大阪`（短縮、`associations.name`）
- 表示: `大阪府クレー射撃協会`（正式、`associations.formal_name`）
- 変換: `lib/affiliation.ts` の `toShortName()` / `toFormalName()`

### 5. ペア大会の概念（大会報告）
同日・同協会のトラップ大会とスキート大会は「ペア」と判定され、`tournament_reports` レコードを1件共有する。どちらの大会管理画面からも同じ報告書を編集可能。
