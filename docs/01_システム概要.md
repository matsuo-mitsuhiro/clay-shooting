# クレー射撃大会 成績管理システム — システム概要

## 目的
クレー射撃大会（トラップ・スキート）の選手登録・点数入力・成績閲覧をリアルタイムで管理するWebアプリケーション。
管理者がタブレット等で点数を入力し、選手・観客がQRコードで成績を閲覧できる。

## 主要画面とURL構造

```
/                         トップページ（大会一覧＋隠し管理ボタン）
/viewer                   閲覧者用：大会選択画面
/viewer/[id]              閲覧者用：大会成績表示
/admin                    管理者用：大会一覧・新規作成
/admin/[id]               管理者用：大会管理（4タブ構成）
  └── 選手登録タブ
  └── 点数登録タブ
  └── 成績確認タブ
  └── 設定タブ
```

## 主要ファイル構成

```
app/
  page.tsx                      トップページ
  viewer/
    page.tsx                    閲覧者：大会一覧
    [id]/page.tsx               閲覧者：成績表示（ViewerPageをimport）
  admin/
    page.tsx                    管理者：大会一覧・作成
    [id]/page.tsx               管理者：大会管理（AdminDetailをimport）
  api/tournaments/
    route.ts                    GET全大会 / POST新規作成
    [id]/route.ts               GET単一 / PUT更新
    [id]/members/route.ts       GET選手一覧 / POST一括保存
    [id]/members/[memberId]/route.ts  GET点数確認 / DELETE選手削除
    [id]/members/copy/route.ts  POST 1日目→2日目コピー
    [id]/scores/route.ts        GET点数 / POST保存
    [id]/results/route.ts       GET成績（v_resultsビュー）
    [id]/reset/route.ts         POST リセット（選手・点数削除）

components/
  admin/
    AdminDetail.tsx             大会管理ページ全体（タブ切替）
    MembersTab.tsx              選手登録タブ
    ScoresTab.tsx               点数登録タブ（テンキーポップアップ）
    ResultsTab.tsx              成績確認タブ
    SettingsTab.tsx             設定タブ（QRコード・大会情報）
  viewer/
    ViewerPage.tsx              閲覧者用成績表示

lib/
  colors.ts                     カラーテーマ定数
  types.ts                      TypeScript型定義
  db.ts                         Neon DB接続

db/
  schema.sql                    DBスキーマ定義（テーブル・ビュー・トリガー）

docs/                           ← このフォルダ（開発資料）
```

## デプロイフロー

```
ローカル編集 → git push origin main → GitHub → Vercel自動デプロイ（約2〜3分）
```

### ローカル開発の注意事項
- Windows環境でEPERMエラーが出る場合は `node windows-eperm-fix.js` を実行
- ただし `next build` はWindowsローカルでは失敗する場合あり（Vercelでは正常にビルドされる）
- TypeScriptチェックは `npx tsc --noEmit --incremental false` で実行可能
