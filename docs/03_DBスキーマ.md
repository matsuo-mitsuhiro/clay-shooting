# データベーススキーマ

## 接続情報
- **クラウドサービス**: Neon（サーバーレスPostgreSQL）
- **プロジェクト名**: Clay-shooting
- **リージョン**: ap-southeast-1（シンガポール）
- **接続方法**: `@neondatabase/serverless` の `neon()` tagged template literal

---

## テーブル構成

### 1. tournaments（大会情報）

| カラム | 型 | NULL | 説明 |
|--------|----|------|------|
| id | SERIAL PK | NO | 自動採番 |
| name | VARCHAR(200) | NO | 大会名 |
| venue | VARCHAR(200) | YES | 射撃場名 |
| day1_date | DATE | YES | 1日目日付 |
| day2_date | DATE | YES | 2日目日付 |
| event_type | VARCHAR(10) | NO | 'trap' or 'skeet'（DEFAULT: 'trap'） |
| day1_set | VARCHAR(100) | YES | 1日目セット番号（例: "1番セット"） |
| day2_set | VARCHAR(100) | YES | 2日目セット番号 |
| admin_qr | TEXT | YES | 管理者用QR（Base64）※現在未使用 |
| viewer_qr | TEXT | YES | 閲覧者用QR（Base64）※現在未使用 |
| created_at | TIMESTAMPTZ | NO | 作成日時 |
| updated_at | TIMESTAMPTZ | NO | 更新日時（トリガーで自動更新） |

---

### 2. members（選手情報）

| カラム | 型 | NULL | 説明 |
|--------|----|------|------|
| id | SERIAL PK | NO | 自動採番 |
| tournament_id | INTEGER FK | NO | tournaments.id（CASCADE DELETE） |
| day | SMALLINT | NO | 1=1日目、2=2日目 |
| group_number | SMALLINT | NO | 組番号（1以上） |
| position | SMALLINT | NO | 組内番号（1〜6） |
| member_code | VARCHAR(20) | YES | 会員番号（半角数字、NULL可） |
| name | VARCHAR(100) | NO | 氏名 |
| belong | VARCHAR(100) | YES | 所属団体 |
| class | CHAR(1) | YES | クラス（A/B/C/D） |
| is_judge | BOOLEAN | NO | 審判資格フラグ（DEFAULT: false） |
| created_at | TIMESTAMPTZ | NO | 作成日時 |

**制約：**
- UNIQUE (tournament_id, day, group_number, position) ← 同一スロット重複禁止
- UNIQUE (tournament_id, day, member_code) ← 同日程内会員番号重複禁止（NULLは除外）

**インデックス：**
- idx_members_tournament_day
- idx_members_tournament_day_group

---

### 3. scores（点数）

| カラム | 型 | NULL | 説明 |
|--------|----|------|------|
| id | SERIAL PK | NO | 自動採番 |
| tournament_id | INTEGER FK | NO | tournaments.id（CASCADE DELETE） |
| member_code | VARCHAR(20) | NO | 会員番号（membersと紐付けキー） |
| name | VARCHAR(100) | YES | 氏名（表示補助用） |
| r1〜r4 | SMALLINT | YES | 1日目ラウンド1〜4（0〜25、NULL=未入力） |
| r5〜r8 | SMALLINT | YES | 2日目ラウンド1〜4（0〜25、NULL=未入力） |
| created_at | TIMESTAMPTZ | NO | 作成日時 |
| updated_at | TIMESTAMPTZ | NO | 更新日時（トリガーで自動更新） |

**制約：**
- UNIQUE (tournament_id, member_code) ← 1大会1選手1レコード

> ⚠️ **重要**: scoresはmember_codeで紐付けるため、member_codeがNULLの選手は点数登録不可

## ビュー：v_results（成績集計）

```
members（day=1） LEFT JOIN members（day=2） LEFT JOIN scores
→ 合計・平均・順位を算出
```

| 出力カラム | 説明 |
|-----------|------|
| day1_total | 1日目合計（r1+r2+r3+r4、NULLは0扱い） |
| day2_total | 2日目合計（r5+r6+r7+r8） |
| total | 総合計 |
| average | 平均（入力済みラウンドのみ、小数1位） |
| rank | RANK()関数（同点同順位、次位スキップ） |

> ⚠️ **既知の問題**: PostgreSQLはNUMERIC集計値を文字列で返す。
> フロントエンドでは `Number(r.average).toFixed(2)` と明示的に変換すること。
> `r.average?.toFixed(2)` は動作しない（文字列にtoFixedメソッドなし）。

## ER図（簡易）

```
tournaments
    │ id
    │
    ├──< members (tournament_id FK)
    │       member_code ─────────────┐
    │                                │
    └──< scores (tournament_id FK)   │
            member_code ─────────────┘
            （member_codeで紐付け、外部キー制約なし）
```
