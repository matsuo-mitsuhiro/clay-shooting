## ビュー：v_results（成績集計）

```sql
day1_members FULL OUTER JOIN day2_members
  → 1日目のみ / 両日 / 2日目のみ の選手すべてを取得
LEFT JOIN scores
  → 点数を結合
RANK() OVER (... 通常選手のみ ...)
  → 順位計算
```

**出力カラム（型は lib/types.ts の `Result` 参照）:**
`tournament_id, member_code, name, belong, class, is_judge, is_non_prize, group1, position, group2, r1〜r8, cb, fr, status, manual_rank, day1_total, day2_total, total, average, rank`

**順位計算ロジック（`status='valid' AND is_non_prize=FALSE` のみ対象）:**
1. `total DESC`
2. `r8 DESC, r7 DESC, ..., r1 DESC`（カウントバック=最終ラウンドからの逆順比較）
3. `cb ASC`（手動カウントバック、小さい方が上位）
4. `fr DESC`（ファイナル、大きい方が上位）

**rank が NULL になる条件:**
- `status='disqualified'`（失格）
- `status='withdrawn'`（棄権）
- `is_non_prize=TRUE`（賞典外）

> ⚠️ **PostgreSQL NUMERIC型の文字列問題**: `average` は文字列で返るので、フロントでは `Number(r.average).toFixed(2)` と明示的に変換すること。`r.average?.toFixed(2)` は動作しない。
