## 1. 競技仕様

### 1.1 種目

| 種目 | DB値 | 表示 |
|------|------|------|
| トラップ | `trap` | トラップ |
| スキート | `skeet` | スキート |

> ダブルトラップは現状未対応（コードに`double_trap`の表示変換だけ存在）

### 1.2 ラウンド構成

| 日程 | ラウンド | DB列 |
|------|---------|------|
| 1日目 | R1〜R4 | r1, r2, r3, r4 |
| 2日目 | R5〜R8 | r5, r6, r7, r8 |

- 1ラウンド: **0〜25点**
- セル表示色: 0〜22は青、**23〜25は赤**（高得点強調）
- テンキーで26以上の入力をブロック

### 1.3 セット番号
- 自由文字列（例: `1番セット`、`10番セット裏`）
- スキート大会では非表示（v3.65.2〜）

## 2. クラス・選手属性

### 2.1 クラス（v3.55〜）

| 値 | 表示 |
|----|------|
| AAA | 最上位クラス（v3.55追加） |
| AA | （v3.10追加、Dを廃止） |
| A | |
| B | |
| C | |

> player_master では `trap_class` と `skeet_class` を**別々に管理**（v3.55〜）。
> 申込時は大会の `event_type` に応じて該当クラスを引く。

### 2.2 審判フラグ
- `is_judge = true` の場合、氏名後ろに **⚑** マーク表示
- 主催協会の審判は組替時に各組へ優先配置

### 2.3 賞典外（is_non_prize、v3.67〜）
- members.is_non_prize、編集は選手管理タブのみ（点数登録タブからは v3.70 で削除）
- 両日参加者は両日分自動同期
- 成績一覧では **rank = NULL**、`── 賞典外 ──` 区切り後に表示
- 大会記録審査表Excelでは摘要列(P)に「賞典外」（カッコなし、v3.76〜）

### 2.4 会員番号
- **半角数字のみ**（全角入力は自動変換）
- 同一大会・同一日程内で重複不可（NULL は除外）
- NULL 可: 会員番号なしでも選手管理に登録可。ただし**点数登録不可**（scoresはmember_codeで紐付け）
- 編集は申込管理タブで可（v3.73〜、選手管理タブからは廃止）
- 編集時は `members`（両日分）/ `scores` / `registrations` の `member_code` を一括同期

## 3. 組・番号構成

- 1組あたり最大 **6名**（position: 1〜6）
- 組数は制限なし（「＋組を追加」で増やせる）
- 1日目・2日目は独立して登録（「1日目からコピー」機能あり）

### 3.1 組替モード
| ボタン | 動作 |
|--------|------|
| 選手組替 | DnDで個別移動。スロット固定・選手データのみ移動 |
| ランダム組替 | 主催協会審判を各組前半・後半に配置 + 両日参加者を前方優先（v3.57〜） |
| 前日の成績順組替 | **2日目専用**。1日目順位順に1組から配置（v3.63〜） |
| 1日目からコピー | day=1 のメンバーを day=2 に複製 |

## 4. 順位計算ルール

### 4.1 通常選手の順位（v_results、v3.43〜）

`status='valid' AND is_non_prize=FALSE` の選手のみ対象、以下の優先順位で `RANK()`:

1. **合計** `total DESC`
2. **カウントバック**（最終ラウンドからの逆順比較）
   - `r8 DESC, r7 DESC, r6 DESC, r5 DESC, r4 DESC, r3 DESC, r2 DESC, r1 DESC`
3. **手動カウントバック** `cb ASC`（1〜6、小さい方が上位）
4. **ファイナルシュートオフ** `fr DESC`（1〜99、大きい方が上位、v3.7〜）

### 4.2 順位 NULL になる条件
- `status='disqualified'`（失格）
- `status='withdrawn'`（棄権）
- `is_non_prize=TRUE`（賞典外）

### 4.3 手動順位（manual_rank）
- 成績確認タブのモーダルで DnD 並び替え
- 設定すると `scores.manual_rank` に保存
- 「自動計算に戻す」で NULL に戻る
- 賞典外選手はモーダルから除外（対象外）

## 5. 申込ルール

### 5.1 申込フロー（v3.55〜現行）
1. **Step 1**: 会員番号 + メール入力 → 6桁コードをメール送信（`registration_tokens.purpose='apply'`）
2. **Step 2**: 6桁コード入力で認証 → フォーム自動補完（player_masterから氏名・所属・クラス）
3. **送信**: 申込確定 → 確認メール送信

### 5.2 参加日（participation_day）
| 値 | 説明 |
|----|------|
| `day1` | 1日目のみ |
| `day2` | 2日目のみ |
| `both` | 両日（2日制大会のみ選択可） |

### 5.3 残数チェック
- 1日目・2日目は **独立カウント**
- `both` は両方に消費
- `max_participants` を超える申込は不可

### 5.4 重複防止
- 同一 `tournament_id × member_code × status='active'` は1件のみ
- 重複時は `registration_logs.log_type='duplicate_error'` 記録

### 5.5 キャンセル
- メール → ワンタイムURL（1時間） → 確認 → `status='cancelled'`, `cancelled_by='user'`
- `cancel_end_at` を過ぎると不可
- 管理者によるキャンセルは `cancelled_by='admin'`

### 5.6 申込→選手管理の移行
- 申込管理タブの「移行」ボタンで `members` テーブルに INSERT
- 移行済みは `transferred_at` で記録
- 申込キャンセル → 移行済みの場合は `members` から自動削除（v3.52〜）
- 選手管理で削除 → active な申込を自動キャンセル（v3.52〜）

### 5.7 連続申込（v3.65〜）
申込完了後、同じメールアドレスで他の大会へ続けて申込可能（再認証なし）。

## 6. ペア大会ルール（大会報告、v3.47〜）

- 同日・同協会のトラップ大会とスキート大会は「ペア」と判定
- `tournament_reports` レコードを **1件共有**（`paired_tournament_id` で双方向参照）
- どちらの大会管理画面からも同じ報告書を編集可能
- 参加人数集計はトラップ＋スキートの合計

## 7. アクセス制御

### 7.1 ロール

| ロール | 認証 | アクセス範囲 |
|-------|------|------------|
| システム管理者 | Google OAuth + `SYSTEM_ADMIN_EMAILS` | 全大会・全機能 |
| 運営管理者 | 会員番号 + bcryptパスワード | 自所属協会の大会のみ |
| 申込者 | 会員番号 + メール（ワンタイムコード） | 自分の申込のみ |
| 閲覧者 | 認証不要 | 公開大会の成績閲覧 |

### 7.2 運営管理者の制約
- `tournament_admins.member_code` → `player_master.member_code` → `player_master.affiliation` で所属判定
- **所属は player_master 側で参照**（引越し対応のため）
- 大会一覧は `tournaments.organizer_cd` が自所属に一致するもののみ
- 点数登録済の大会の **削除は不可**（リセット・編集は可）
- 運営管理者の作成・削除はシステム管理者のみ可

### 7.3 招待トークン
- システム管理者が `/admin/admins` で発行 → 招待URL（`/admin/register?token=...`）をメールまたはQRで配布
- `admin_invitations` テーブル、有効期限付き、1回使い切り

### 7.4 認証廃止履歴
- v3.78 でトップページ左下の隠しボタン+`repros`プロンプト方式は削除済
- 現在は NextAuth.js v4 + middleware.ts の正規認証のみ

## 11. Excel出力ルール

### 11.1 出力方式（v3.35〜）
- **JSZip + XML 直接操作方式**（ExcelJS は廃止）
- テンプレート: `templates/inspection-report-template.xlsx`、`templates/report-template.xlsx`
- 数式・printerSettings・pageSetup・ロゴをすべて保持
- 数式キャッシュ（calcChain）クリア必須（テンプレートのサンプルデータ残存防止）

### 11.2 大会記録審査表Excel
- 72行（2ページ）テンプレート
- 改ページは Row 45 に rowBreaks 設定済
- 選手不在行は A〜P列すべて空欄化（v3.76〜）
- 摘要列(P)に `賞典外`/`失格`/`棄権`/`CBn`/`FRn` をスペース区切りで併記

### 11.3 大会報告書Excel
- ペア大会（トラップ+スキート）を1ファイルに集約
- 未保存状態でのExcel出力はブロック（アラート表示）
- ④⑤ 本部納入金（O31, Q31, O32, Q32）は画面計算値を直接書き込み（v3.76〜）

## 12. データリセット

「危険ゾーン」（大会設定タブ）から実行:
- 対象: `members` + `scores` の全削除
- 大会情報・QRコード・申込データは**保持**
- 実行者は `tournaments.reset_by` / `reset_at` に記録
- 認証は現在ログイン中の管理者本人で代替（v3.78〜、旧 `repros` 方式廃止）
